================================================================================
POCKET SHOP — BACKEND DATA MODELS
================================================================================

All models live in pocket_backend/. Each app has its own models.py.
Database: PostgreSQL (production), SQLite3 (development).

--------------------------------------------------------------------------------
Model: Order
--------------------------------------------------------------------------------
FIELD                       TYPE            NOTES
order_number                CharField       Unique. Auto-generated on save (e.g. PS-20240115-XXXX)
buyer                       ForeignKey      User (role=buyer)
seller                      ForeignKey      User (role=seller)
total_price                 DecimalField    Sum of items (ZMW)
delivery_fee                DecimalField    Quoted at checkout
fulfillment_type            CharField       delivery / pickup
status                      CharField       See status flow below
delivery_address            TextField
delivery_lat                DecimalField    GPS coordinates of delivery address
delivery_lng                DecimalField
special_instructions        TextField
payment_method              ForeignKey      BuyerPaymentMethod — nullable
payment_provider_snapshot   CharField       Snapshot of provider at order time
payment_account_snapshot    CharField       Snapshot of phone number at order time

Properties:
  grand_total   total_price + delivery_fee

ORDER STATUS FLOW:
  pending
    → payment_pending  (payment initiated but not yet confirmed)
    → accepted         (payment confirmed by PawaPay webhook)
    → preparing        (seller has started preparing)
    → out_for_delivery (rider picked up, in transit)
    → delivered        (QR dropoff confirmed)
    → cancelled        (payment failed, timeout, or user/seller cancelled)

  subtotal   price × quantity

--------------------------------------------------------------------------------
Model: RefundRequest  (one per order)
--------------------------------------------------------------------------------
FIELD           TYPE            NOTES
order           OneToOneField   Order
requested_by    ForeignKey      User (buyer)
reason          TextField
status          CharField       See refund status flow below
seller_note     TextField       Seller's response
admin_note      TextField       Admin's final decision

REFUND STATUS FLOW:
  pending_seller
    → approved_by_seller   / rejected_by_seller
    → escalated            (buyer escalates rejected refund to admin)
    → approved_by_admin    / rejected_by_admin
    → refunded             (money returned to buyer's mobile money)

--------------------------------------------------------------------------------
Model: OrderRating  (multi-party ratings)
--------------------------------------------------------------------------------
FIELD       TYPE            NOTES
order       ForeignKey      Order
author      ForeignKey      User (who is giving the rating)
target_role CharField       buyer / seller / delivery
score       IntegerField    1–5
comment     TextField       Optional

Constraint: Unique on (order, author, target_role)

Rating directions:
  Buyer rates → rider experience (target_role=delivery)
  Buyer rates → overall order experience (target_role=seller)
  Rider rates → seller readiness (target_role=seller)

--------------------------------------------------------------------------------
Model: DeliveryPricingConfig  (singleton — only one row)
--------------------------------------------------------------------------------
FIELD                           TYPE            NOTES
per_km_rate                     DecimalField    ZMW per km
short_distance_threshold_km     DecimalField    Below this → flat rate applies
short_distance_flat_rate        DecimalField    ZMW flat rate for short trips

Logic:
  if distance_km <= threshold → fee = short_distance_flat_rate
  else                        → fee = per_km_rate × distance_km

--------------------------------------------------------------------------------
FIELD               TYPE            NOTES

APP: payments
================================================================================

--------------------------------------------------------------------------------
Model: Transaction
--------------------------------------------------------------------------------
FIELD               TYPE            NOTES
transaction_id      UUIDField       Primary key. UUID4 generated on creation
order               ForeignKey      Order
transaction_type    CharField       deposit / payout / refund
amount              DecimalField    ZMW
currency            CharField       Always 'ZMW'
provider            CharField       MTN_MOMO_ZMB / AIRTEL_OAPI_ZMB / ZAMTEL_MONEY_ZMB
payer_number        CharField       Phone number that made the payment
recipient           ForeignKey      User — nullable (for payouts and refunds)
recipient_role      CharField       seller / delivery / buyer
payout_stage        CharField       See stages below
trigger_event       CharField       manual / pickup_qr / dropoff_qr / order_cancelled
payout_method       CharField       manual / gateway
failure_message     TextField       PawaPay error message on failure
status              CharField       pending / accepted / completed / failed / duplicate_ignored

PAYOUT STAGE VALUES:
  na                      Not a payout (deposit or refund)
  pickup_pending_scan     Waiting for pickup QR scan
  dropoff_pending_scan    Waiting for dropoff QR scan
  ready_for_payout        All conditions met; payout queued
  payout_sent             Sent to PawaPay payout API or manual
  payout_paid             Confirmed received by recipient
  payout_failed           PawaPay returned error on payout

No wallet balance is tracked — this is a pure transaction ledger.
