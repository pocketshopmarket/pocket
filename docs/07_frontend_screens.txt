================================================================================
POCKET SHOP — FLUTTER FRONTEND: SCREENS & NAVIGATION
================================================================================

App Name:     Pocket Shop
Framework:    Flutter 3.10.8
Router:       GoRouter (go_router package)
Entry Point:  lib/main.dart
Router File:  lib/router/app_router.dart

--------------------------------------------------------------------------------
APP STARTUP SEQUENCE
--------------------------------------------------------------------------------

1. main.dart initializes:
   - Firebase (firebase_init.dart → initFirebaseIfSupported)
   - FlutterScreenUtil (responsive sizing)
   - Riverpod ProviderContainer
   - GoRouter with authInitializationProvider listener

2. App navigates to /splash (SplashScreen)

3. SplashScreen reads tokens from SharedPreferences:
   - If tokens found → calls GET /api/auth/profile/
   - If profile loads → redirects to role home (buyer/seller/delivery)
   - If 401 → tries refresh → if still fails → clears session → /phone
   - If no tokens → /phone

4. FCM token registered after successful auth:
   POST /api/auth/fcm-token/ with device FCM token

--------------------------------------------------------------------------------
NAVIGATION ARCHITECTURE
--------------------------------------------------------------------------------

GoRouter uses a StatefulShellRoute for each role's main navigation.
StatefulShellRoute preserves scroll position and state when switching tabs.

PUBLIC ROUTES (before auth):
  /splash              SplashScreen
  /phone               PhoneLoginScreen
  /signup              SignupScreen
  /otp                 OtpVerificationScreen
  /role-selection      RoleSelectionScreen
  /forgot-password     ForgotPasswordScreen
  /reset-password      ResetPasswordScreen

AUTHENTICATED ROUTES:
  Redirect based on User.role after login:
    buyer    → /buyer/home
    seller   → /seller/dashboard
    delivery → /delivery/home
    admin    → /admin

BUYER SHELL (5 tabs, persistent bottom nav):
  Tab 0: Home         /buyer/home
  Tab 1: Shop         /buyer/shop
  Tab 2: Wishlist     /buyer/wishlist
  Tab 3: Cart         /buyer/cart
  Tab 4: Profile      /buyer/profile

SELLER SHELL (4 tabs):
  Tab 0: Dashboard    /seller/dashboard
  Tab 1: Products     /seller/products
  Tab 2: Orders       /seller/orders
  Tab 3: Shop         /seller/shop  (placeholder / own shop view)

DELIVERY SHELL (4 tabs):
  Tab 0: Home         /delivery/home
  Tab 1: Active       /delivery/active
  Tab 2: Earnings     /delivery/earnings
  Tab 3: Account      /delivery/account

STANDALONE ROUTES (no bottom nav):
  /delivery/offers     DeliveryOffersScreen    (modal-like over shell)
  /notifications       NotificationsScreen     (accessible from all roles)
  /refund-requests     RefundRequestsScreen    (accessible from all roles)
  /admin               AdminDashboardScreen

NESTED UNDER BUYER HOME (push navigation within tab):
  /buyer/product-details?id=<id>     BuyerProductDetailsScreen
  /buyer/search                      BuyerSearchScreen
  /buyer/track-order?order=<number>  OrderTrackingScreen
  /buyer/orders                      BuyerOrdersScreen
  /buyer/orders/:id                  BuyerOrderDetailScreen
  /buyer/payment-pending?tx=<uuid>   PaymentPendingScreen
  /buyer/add-payment-method          AddPaymentMethodScreen

NESTED UNDER SELLER PRODUCTS:
  /seller/products/add               AddProductScreen
  /seller/products/edit?id=<id>      EditProductScreen
  /seller/products/:id/reviews       SellerProductReviewsScreen
  /seller/payout-methods             SellerPayoutMethodsScreen
  /seller/payout-history             SellerPayoutHistoryScreen

SCREEN INVENTORY — AUTH
--------------------------------------------------------------------------------

SplashScreen                (auth/screens/splash_screen.dart)
  - Shows app logo
  - Initializes session (tokens + profile check)
  - Auto-redirects in ~2 seconds

PhoneLoginScreen            (auth/screens/phone_login_screen.dart)
  - Phone number input with Zambia +260 prefix

  - Navigates to OtpVerificationScreen

SignupScreen                (auth/screens/signup_screen.dart)

  - Role selection (buyer/seller/delivery)
  - Terms agreement

OtpVerificationScreen       (auth/screens/otp_verification_screen.dart)
  - 6-digit OTP pin input
  - Countdown timer (10 min expiry)
  - Resend OTP option
  - Submits to verifyOTP → handles is_new_user + role redirect

RoleSelectionScreen         (auth/screens/role_selection_screen.dart)
  - Shown when user doesn't have a role yet
  - Three tiles: Buyer / Seller / Delivery Agent

ForgotPasswordScreen        (auth/screens/forgot_password_screen.dart)
  - Phone number input → triggers OTP for reset

ResetPasswordScreen         (auth/screens/reset_password_screen.dart)
  - OTP + new password input

ChangePasswordScreen        (auth/screens/change_password_screen.dart)
  - Old password + new password (for logged-in users)

THEMING (app_theme.dart)
--------------------------------------------------------------------------------

Theme:    Material3 Light

Screen sizing: flutter_screenutil

  Base design size: typically 375×812 (standard phone)
  Size constants: lib/core/constants/app_sizes.dart

--------------------------------------------------------------------------------
RESPONSIVE / PLATFORM
--------------------------------------------------------------------------------

The app targets Android, iOS, and Windows Desktop.
flutter_secure_storage is DISABLED on Windows Desktop due to compatibility.
SharedPreferences is used for all platforms (tokens, user session).

flutter_screenutil handles responsive sizing across different screen sizes.
