--------------------------------------------------------------------------------
ARCHITECTURE OVERVIEW
--------------------------------------------------------------------------------

State Management:  Riverpod (flutter_riverpod)
API Client:        Dio (http client with interceptors)
Local Storage:     SharedPreferences (tokens, user session)
Pattern:           StateNotifier for mutable state, Provider/FutureProvider for
                   read-only or async state

Layer structure:
  Screen → Provider/Notifier → Service → API (Dio) → Backend

  Screens read state from Riverpod providers and call notifier methods.
  Notifiers call service methods.
  Services make HTTP calls via ApiService (Dio client).
  State updates propagate back to screen via Riverpod's reactive model.

--------------------------------------------------------------------------------
API SERVICE  (lib/services/api_service.dart)
--------------------------------------------------------------------------------

Singleton Dio client. All services use this client.

Configuration:
  baseUrl         From AppConstants.baseUrl
  connectTimeout  30,000 ms
  receiveTimeout  30,000 ms

Interceptors:
  REQUEST interceptor:
    - Reads access_token from SharedPreferences
    - Adds Authorization: Bearer <token> header to every request

  RESPONSE interceptor:
    - On HTTP 401:
      1. Reads refresh_token from SharedPreferences
      2. Makes POST /api/auth/refresh/ (without auth interceptor to avoid loop)
      3. On success: saves new access_token + refresh_token to SharedPreferences,
                     retries the original failed request
      4. On failure: clears both tokens from SharedPreferences,
                     emits auth failure event (triggers navigation to /phone)

Methods:
  get(path, {queryParameters})
  post(path, {data, isFormData})
  put(path, {data})
  patch(path, {data})
  delete(path)

Error handling:
  DioException → wraps with descriptive error message
  Network errors → "No internet connection"
  Timeout errors → "Request timed out"
  Server errors → passes through response body for error display

AUTH PROVIDER  (lib/providers/auth_provider.dart + auth.dart)
--------------------------------------------------------------------------------

authProvider  →  StateNotifierProvider<AuthNotifier, AuthState>

AuthState fields:
  user           User? (null when not authenticated)
  isLoading      bool
  error          String?
  isAuthenticated bool
  isInitialized  bool  (True after startup auth check completes)

AuthNotifier methods:
  initialize()         Called on app start. Loads user from SharedPreferences.
                       If token found, fetches /api/auth/profile/ to verify.
  login(phone, pass)   Calls AuthService.login. Saves tokens + user.
  sendOtp(phone)       Calls AuthService.sendOTP.
  verifyOtp(...)       Calls AuthService.verifyOTP. Saves tokens + user.
  logout()             Calls AuthService.logout. Clears storage. Resets state.
  refreshUser()        Re-fetches /api/auth/profile/ and updates state.
  setAuthenticatedUser(user, access, refresh)  Used after verify-otp.

authInitializationProvider  →  FutureProvider<void>
  Triggers AuthNotifier.initialize() once on app start.
  The GoRouter listens to authProvider.isInitialized to decide initial route.

authServiceProvider  →  Provider<AuthService>
  Exposes AuthService instance. Used by AuthNotifier.

CART PROVIDER  (lib/providers/cart_provider.dart)
--------------------------------------------------------------------------------

cartProvider  →  StateNotifierProvider<CartNotifier, CartState>

CartState:
  items      List<CartItem>
  isLoading  bool
  error      String?

Methods:
  loadCart()               GET /api/orders/cart/
  addItem(productId, qty, variantId)  POST /api/orders/cart/
  updateItem(itemId, qty)  PATCH /api/orders/cart/<id>/
  removeItem(itemId)       DELETE /api/orders/cart/<id>/
  clearCart()              Local state clear after order placement

Computed:
  totalItems   Sum of all CartItem.quantity
  totalPrice   Sum of all CartItem subtotals

The cart tab badge in BuyerMainScreen reads cartProvider.totalItems.

SERVICE CLASSES
--------------------------------------------------------------------------------

Each service class is a thin HTTP client layer — no state management.
Injected into providers via serviceProvider instances.

AuthService       (lib/services/auth_service.dart)
  initialize()    Load tokens + user from SharedPreferences
  sendOTP(phone)
  verifyOTP(phone, code, role, name)
  login(phone, password)
  getCurrentUser()
  logout()
  saveFcmToken(token)

ProductService    (lib/services/product_service.dart)
  getProducts(filters)
  getProductDetail(id)
  searchProducts(query)
  getRecommended()
  getBanners()
  trackInteraction(productId, type)

OrderService      (lib/services/order_service.dart)
  getCart()
  addToCart(productId, quantity, variantId)
  updateCartItem(itemId, quantity)
  removeCartItem(itemId)
  createOrder(params)
  getOrders()
  getOrderDetail(id)
  cancelOrder(id)
  acceptOrder(id)
  rateOrder(id, targetRole, score, comment)
  requestRefund(id, reason)

DeliveryService   (lib/services/delivery_service.dart)
  getAvailableOrders()
  acceptDelivery(orderId)
  updateLocation(lat, lng)
  getActiveAssignment()
  loadTrackingData(orderNumber)
  getOffers()
  generateHandoffToken(assignmentId, step)
  verifyHandoffToken(token, orderId, step)
  getDeliveryQuote(address, lat, lng)

CategoryService   (lib/services/category_service.dart)
  getCategories()

ReviewService     (lib/services/review_service.dart)
  getReviews(productId)
  submitReview(productId, rating, comment)

NotificationService (lib/services/notification_service.dart)
  getNotifications()
  getUnreadCount()
  markRead(id)
  markAllRead()

FirebaseInit      (lib/services/firebase_init.dart)
  initFirebaseIfSupported()
    - Initializes Firebase Core
    - Gets FCM token
    - Sets up onMessage (foreground notification handler)
    - Sets up onMessageOpenedApp (tapped notification handler)
    - Sets up background message handler
    - Calls POST /api/auth/fcm-token/ to register token with backend
