CLOUDINARY CONFIGURATION
--------------------------------------------------------------------------------

Why: Railway filesystem is ephemeral — files are lost on redeploy.
Cloudinary provides persistent cloud media storage.

Setup:
  1. Create free Cloudinary account at cloudinary.com
  2. Find CLOUDINARY_URL in dashboard: cloudinary://api_key:api_secret@cloud_name
  3. Set in Railway environment variables

Django will automatically use cloudinary_storage when CLOUDINARY_URL is present.
Without it, media files go to local /media/ directory (fine for development).

Folder structure in Cloudinary:
  pocket_shop/profile_photos/
  pocket_shop/product_images/
  pocket_shop/verification_docs/

--------------------------------------------------------------------------------
CORS CONFIGURATION
--------------------------------------------------------------------------------

Package: django-cors-headers
Middleware: corsheaders.middleware.CorsMiddleware (must be FIRST in MIDDLEWARE)

In development:
  CORS_ALLOW_ALL_ORIGINS = True  (or set in settings for dev)

In production:
  CORS_ALLOWED_ORIGINS should list only known client origins.
  The Flutter mobile app makes requests directly to the API — CORS affects
  only browser-based clients (the landing page / admin access).

--------------------------------------------------------------------------------

Used for: OTP delivery via SMS

  AFRICAS_TALKING_API_KEY    From Africa's Talking dashboard

         Test at: https://simulator.africastalking.com/
